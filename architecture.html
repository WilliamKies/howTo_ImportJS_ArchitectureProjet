Architecture & structure de projet

Qu'est-ce qu'une bonne structure/architecture de projet ?

En développement web, une architecture de projet fait référence à la manière dont vous organisez et structurez votre code, vos fichiers et vos dossiers pour construire une application web. Cela implique de prendre des décisions concernant la disposition des fichiers, la séparation des préoccupations, la gestion des dépendances, etc. L'objectif principal d'une architecture de projet est de créer une base solide pour votre application, facilitant ainsi son développement, sa maintenance et son évolutivité.

Les aspects importants

  1. Organisation des fichiers & dossiers : Déterminez comment organiser vos fichiers et dossiers pour votre projet. Cela peut inclure des dossiers pour les actifs statiques (comme les images, les styles et les scripts), le code source de l'application (comme les composants, les pages, les services), les tests, les configurations, etc.
  2. Séparation des préoccupations : Assurez-vous de séparer les différentes parties de votre application de manière logique et modulaire. Par exemple, séparez le code HTML, CSS et JavaScript, séparez les composants réutilisables, les pages spécifiques de l'application, les fonctionnalités communes, etc.
  3. Gestion des dépendances : Déterminez comment gérer les dépendances de votre projet, que ce soit en utilisant des gestionnaires de paquets comme npm ou Yarn, en reliant des bibliothèques externes via des CDN, ou en incluant des fichiers localement dans votre projet.
  4. Modularité : Favorisez la modularité de votre code en le divisant en petits morceaux réutilisables. Cela rend votre code plus facile à comprendre, à tester et à maintenir, et favorise la réutilisation du code dans différentes parties de votre application.
  5. Extensibilité & évolutivité : Concevez votre architecture de projet de manière à ce qu'elle puisse évoluer et s'adapter facilement aux besoins changeants de votre application. Cela implique de choisir des structures et des conventions qui permettent à votre projet de grandir sans introduire trop de complexité ou de rigidité.

Exemples d'architectures

Architecture simple

Dans cette architecture, tous les fichiers sont organisés de manière minimale, ce qui conviens aux petits projets.

  • my-project/
    • ├── index.html
    • └── assets/
      • ├── images/
      • ├── pages/
      • ├── styles/
      • └── scripts/

dans cette structure, le fichier index.html est à la racine du projet et les actifs statiques (images, styles, scripts) sont regroupés dans le dossier `assets/`. Cette architecture est simple et facile à comprendre, mais elle peut devenir encombrée pour les projets plus importants.

Architecture complexe

Dans cette architecture, le code est organisé de manière modulaire, ce qui facilite la maintenance et l'extensibilité du projet.

Dans cette structure, le code source de l'application est organisé de manière plus modulaire. Les composants sont regroupés dans le dossier `components/`, les pages dans le dossier `pages/`, les styles dans le dossier `styles/`, et les scripts JavaScript dans le dossier `scripts/`.

  • my-project/
    • ├── index.html
    • ├── data.json
    • ├── assets/
      • ├── images/
      • ├── styles/
        • ├── style.css
        • ├── bootstrap.min.css
      • └── scripts/
        • ├── jquery.min.js
        • ├── bootstrap.bundle.min.js
        • ├── main.js
        • ├── theme-switcher.js
    • ├── src/
      • ├── components/
          • // script JS permettant de générer dynamiquement le header HTML sur toute les pages
        • ├── header.js
        • └── footer.js
      • ├── pages/
        • ├── services.html
        • └── about.html
      • ├── styles/
        • ├── services.css
        • └── about.css
      • └── scripts/
        • ├── form-validation.js
